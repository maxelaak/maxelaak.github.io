# Portfolio Website - Student GitHub Practice

## Overview

I created this repository during the development of my personal portfolio website to showcase my work and practice using GitHub. The website is built using **HTML** and **CSS**, with a focus on clean, modern design and responsive layouts. Markdown was also used to format this README section.

The project is intended to demonstrate my ability to utilize **GitHub** for version control and as a platform to display work for professional purposes.

## Purpose

The primary goals of this project are:

- **Practice using GitHub**: Learn how to create, commit, and manage a GitHub repository while making changes to a project.
- **Create a Professional Portfolio**: Develop a personal portfolio website to showcase skills, projects, and accomplishments.
- **Improve HTML & CSS Skills**: Enhance HTML and CSS skills while building a visually appealing and responsive web page.
- **Version Control**: Learn how to use Git and GitHub for version control, branching, and collaboration.

## Project Features

- **Home Page**: A welcome section introducing the student and the purpose of the portfolio.
- **About Section**: A brief bio and professional background, including skills and interests.
- **Projects Section**: A showcase of completed projects, each with descriptions and links.
- **Responsive Design**: The website adjusts to different screen sizes (desktops, tablets, and phones) for a seamless experience.

## Technologies Used

- **HTML5**: Used to structure the content and create the various sections of the portfolio.
- **CSS3**: Used for styling the website and ensuring it has a modern, responsive design.
- **Git & GitHub**: Utilized for version control and collaboration throughout the development process.
- **Markdown**: Formatting language used in GitHub README sections.
- **Google Fonts**: Used for custom typography to enhance the readability and design of the site.
